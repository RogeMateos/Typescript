Arrays

Problem

In JavaScript, an array can contain mixed types (numbers, strings, objects) causing unexpected errors.

Solution

TypeScript allows defining typed arrays: all elements must be of the specified type.

Visual Analogy

"A bookshelf only contains books, you don't mix them with shoes. A typed array only contains elements of the declared type."

// Two ways to declare typed arrays
let numbers: number[] = [1, 2, 3, 4, 5];
let names: Array<string> = ["Anna", "John", "Peter"];

// TypeScript infers the type
let ages = [25, 30, 35]; // number[]

// numbers.push("text"); // ❌ Error: string is not number
let fruits: string[] = ["apple", "pear"];

// Common methods with type safety
fruits.push("grape"); // ✅ OK
fruits.map(f => f.toUpperCase()); // ✅ TS knows f is string
fruits.filter(f => f.length > 4); // ✅ Correct autocomplete

// Array of objects
let users: { name: string; age: number }[] = [
  { name: "Anna", age: 25 }
];

Best Practices

  • Prefer number[] over Array<number> for readability
  • Define interfaces for arrays of complex objects
  • Use readonly for immutable arrays: readonly number[]

Interview Prep

Q: What is the difference between number[] and Array<number>?

A: There is no functional difference, they are equivalent syntax. number[] is more concise and preferred by the community, while Array<number> is the generic syntax. Both define an array of numbers.
