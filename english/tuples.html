Tuples: Arrays with Fixed Structure

Problem

Arrays allow any number of elements of the same type, but sometimes you need a fixed number of elements with specific types in each position.

Solution

Tuples are arrays with a fixed number of elements where each position has a specific predefined type.

Visual Analogy

"Like GPS coordinates: they're always two numbers in fixed order [latitude, longitude]. You can't swap them or add a third value without changing their meaning."

// Defining a Tuple
let user: [string, number, boolean];

// Correct assignment
user = ["Anna", 25, true];

// ❌ Error: order matters
// user = [25, "Anna", true];

// Access by index (with correct types)
let name: string = user[0];   // ✅ OK
let age: number = user[1];    // ✅ OK
let active: boolean = user[2]; // ✅ OK
// Real use case: API Response
type ApiResponse = [number, string, any];

function fetchData(): ApiResponse {
  return [200, "OK", { data: "..." }];
}

const [code, message, data] = fetchData();
// code is number
// message is string
// data is any

// Tuple with labels (TS 4.0+)
type Coordinates = [lat: number, lng: number];
let position: Coordinates = [40.7128, -74.0060];

Best Practices

  • Use tuples for related values with fixed order (like coordinates or RGB)
  • Limit tuples to 2-3 elements; for more, use interfaces
  • Use readonly for immutable tuples: readonly [string, number]
  • Add labels for clarity: [name: string, age: number]

Interview Prep

Q: What is the difference between a Tuple and an Array in TypeScript?

A: An Array has variable length and all elements are the same type (number[]). A Tuple has fixed length and each position can have a different type ([string, number]). Tuples are ideal when the order and number of elements have semantic meaning.
