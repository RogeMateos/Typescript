Required Parameters

Problem

Passing an incorrect number of arguments causes unpredictable errors.

Solution

TypeScript requires all parameters to be passed, catching errors at compile time.

function calculateArea(width: number, height: number): number {
  return width * height;
}

calculateArea(5, 10); // ‚úÖ Correct
calculateArea(5); // ‚ùå Error: missing 'height'

Summary

  • ‚Ä¢In TypeScript, all parameters are required by default
  • ‚Ä¢A value must be passed for each parameter
  • ‚Ä¢Order matters
  • ‚Ä¢TypeScript checks this at compile time, not runtime

1 What are optional parameters

An optional parameter is one that may or may not be provided when calling a function. They are marked with the ? symbol:

function activatePower(name: string, lastName?: string) {
  const fullName = `${name} ${lastName}`;
  console.log(`${fullName} activated!`);
}

// Usage
activatePower("Tony");
activatePower("Tony", "Stark");

üëâ TypeScript allows both calls because lastName is optional.

2 What does ? really mean in TypeScript

When you write:

lastName?: string

TypeScript internally interprets it as:

lastName: string | undefined
  • ‚Ä¢The parameter may not exist
  • ‚Ä¢If it exists, it can be undefined

‚ö†Ô∏è This is key: It forces you to handle that case.

3 Handling optional values correctly

‚ùå Common mistake:

lastName.toUpperCase(); // ‚ùå Can break if undefined

‚úÖ Correct ways:

lastName?.toUpperCase();

if (lastName) {
  lastName.toUpperCase();
}

const safeLastName = lastName ?? "- no lastName -";

4 Defaults: || vs ?? vs default parameters

‚ùå Dangerous use of ||:

const fullName = `${name} ${lastName || "- no lastName -"}`;
// Problem: || treats falsy values like "", 0 as missing

‚úÖ Better: Nullish coalescing (??):

const fullName = `${name} ${lastName ?? "- no lastName -"}`;
// Only applies default if null or undefined

‚úÖ Even better: default parameters:

function activatePower(
  name: string,
  lastName: string = "- no lastName -"
) {
  console.log(`${name} ${lastName} activated!`);
}
// Clearer, more readable, and explicit in API

5 Fundamental rule: optionals go at the end

‚ùå Incorrect:

function f(a?: string, b: number) {}

‚úÖ Correct:

function f(b: number, a?: string) {}
// If you need many optionals, use Options Object

6 Professional pattern: Options Object

This is the most common pattern in real projects:

type ActivatePowerOptions = {
  name: string;
  lastName?: string;
  shout?: boolean;
};

function activatePower({
  name,
  lastName,
  shout = false,
}: ActivatePowerOptions) {
  const fullName = `${name} ${lastName ?? "- no lastName -"}`;
  const message = `${fullName} activated!`;
  console.log(shout ? message.toUpperCase() : message);
}

// Usage
activatePower({ name: "Tony" });
activatePower({ name: "Tony", lastName: "Stark" });
activatePower({ name: "Tony", shout: true });

Why this pattern is key:

  • ‚Ä¢ No dependency on parameter order
  • ‚Ä¢ Scales without breaking existing code
  • ‚Ä¢ Very readable
  • ‚Ä¢ Ideal for APIs and React

7 Optional parameters in React (Props)

Required vs optional props:

type UserCardProps = {
  name: string;
  lastName?: string;
};

export function UserCard({ name, lastName }: UserCardProps) {
  return (
    <div>
      <h3>
        {name} {lastName ?? ""}
      </h3>
    </div>
  );
}

Modern React defaults:

type BadgeProps = {
  label: string;
  variant?: "success" | "warning" | "danger";
};

export function Badge({
  label,
  variant = "success",
}: BadgeProps) {
  return <span>{label} ({variant})</span>;
}
// ‚úÖ Use defaults in destructuring
// ‚úÖ Avoid defaultProps (old pattern)

8 Optional parameters in real APIs

Typical case: optional filters in a search.

type SearchUsersParams = {
  query: string;
  page?: number;
  pageSize?: number;
};

async function searchUsers({
  query,
  page = 1,
  pageSize = 20,
}: SearchUsersParams) {
  const url = new URL("/api/users", window.location.origin);

  url.searchParams.set("query", query);
  url.searchParams.set("page", String(page));
  url.searchParams.set("pageSize", String(pageSize));

  const response = await fetch(url.toString());
  if (!response.ok) throw new Error("Request failed");

  return response.json();
}

üí° The function always works with safe values, even if the user doesn't pass all parameters.
