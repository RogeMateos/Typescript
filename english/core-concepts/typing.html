The Typing Problem

Problem

JavaScript is dynamic. You can assign a number to a variable that used to be text, and the program won't complain until it crashes in production.

Solution

TypeScript adds 'Static Typing'. It catches errors while you write code, not while the user browses.

Visual Analogy

"Think of labeling boxes during a move. Without labels, you have to open everything to know what's inside (JS). With labels, you know exactly what to expect without opening them (TS)."

// JavaScript (Dangerous)
function add(a, b) {
  return a + b;
}
console.log(add('5', 5)); // Result: '55'
// TypeScript (Safe)
function add(a: number, b: number): number {
  return a + b;
}
// Error: Argument of type 'string' is not assignable...

Interview Prep

Q: What is TypeScript and why use it?

A: It is a JS superset that adds static typing, allowing error detection during compile time and improving autocompletion.
