Function Type

Problem

How do you type a variable that stores a function?

Solution

Define the exact type: what parameters it accepts and what it returns.

Visual Analogy

"A function is like a machine in a box. The box exactly tells what goes in and what comes out."

// Type for a function that takes two numbers and returns a number
type Operation = (a: number, b: number) => number;

const add: Operation = (a, b) => a + b;
const subtract: Operation = (a, b) => a - b;

add(5, 3); // ‚úÖ

Interview Prep

Q: Why type functions as values?

A: To ensure that callbacks, handlers, and functions passed as arguments always have the correct shape.

Best Practices Checklist

  • Don't use any for functions
  • Type variables that store callbacks
  • Reuse signatures with type
  • Use generics when the type varies

1 The Problem: Functions Without Type

This is dangerous:

let myFunction: any;

myFunction = 10;
myFunction = 'hello';
myFunction = () => 'ok';

‚ùå any destroys safety

A function can be anything, and that causes bugs

2 The Correct Function Signature 

let operation: (a: number, b: number) => number;

operation = (x, y) => x + y; // ‚úÖ OK
operation = (x, y) => x * y; // ‚úÖ OK
operation = (x, y) => `${x} + ${y}`; // ‚ùå Error: returns string

‚úî What is this

Define exactly what goes in and what comes out. It's a contract.

3 Reuse Signatures with Type

Better than repeating the signature:

type Operation = (a: number, b: number) => number;

const sum: Operation = (a, b) => a + b;
const mul: Operation = (a, b) => a * b;
const div: Operation = (a, b) => a / b;

Advantages:

  • ‚Ä¢ More readable code
  • ‚Ä¢ Reusable across multiple functions
  • ‚Ä¢ Scalable

Real Example: React Props 

type OnSave = (value: string) => void;

function Form({ onSave }: { onSave: OnSave }) {
  return (
    <button onClick={() => onSave('data')}>
      Save
    </button>
  );
}

5 Consume APIs with Typed Mappers 

type Mapper<TIn, TOut> = (input: TIn) => TOut;

type ApiUser = { id: number; first_name: string };
type UiUser = { id: number; name: string };

const mapUser: Mapper<ApiUser, UiUser> = (u) => ({
  id: u.id,
  name: u.first_name,
});

Powerful Generic Pattern

Decouples backend from frontend. API changes = just update mapper

6 Strategy Pattern with Functions 

type SortStrategy<T> = (a: T, b: T) => number;

const sortByName: SortStrategy<{ name: string }> = (a, b) =>
  a.name.localeCompare(b.name);

const sortByAge: SortStrategy<{ age: number }> = (a, b) =>
  a.age - b.age;

Advantage

Changes behavior without nested if/else. Cleaner code.
