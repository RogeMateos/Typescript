Arrays

Problem

In JavaScript, an array can contain mixed types (numbers, strings, objects) causing unexpected errors.

Solution

TypeScript allows defining typed arrays: all elements must be of the specified type.

Visual Analogy

"A bookshelf only contains books, you don't mix them with shoes. A typed array only contains elements of the declared type."

// Two ways to declare typed arrays
let numbers: number[] = [1, 2, 3, 4, 5];
let names: Array<string> = ["Anna", "John", "Peter"];

// TypeScript infers the type
let ages = [25, 30, 35]; // number[]

// numbers.push("text"); // ‚ùå Error: string is not number
let fruits: string[] = ["apple", "pear"];

// Common methods with type safety
fruits.push("grape"); // ‚úÖ OK
fruits.map(f => f.toUpperCase()); // ‚úÖ TS knows f is string
fruits.filter(f => f.length > 4); // ‚úÖ Correct autocomplete

// Array of objects
let users: { name: string; age: number }[] = [
  { name: "Anna", age: 25 }
];

Best Practices

  • Prefer number[] over Array<number> for readability
  • Define interfaces for arrays of complex objects
  • Use readonly for immutable arrays: readonly number[]

Interview Prep

Q: What is the difference between number[] and Array<number>?

A: There is no functional difference, they are equivalent syntax. number[] is more concise and preferred by the community, while Array<number> is the generic syntax. Both define an array of numbers.

1. The Real Problem in JavaScript

In plain JavaScript, arrays can contain anything:

// JavaScript - Chaos
const mixed = [42, "text", true, null, { name: "Ana" }];
const numbers = [1, 2, 3];
numbers.push("oops"); // No error!

// Then this crashes:
const total = numbers.reduce((a, b) => a + b); // NaN üí•

‚ö†Ô∏è The error appears at runtime, not when writing code.

2. Basic Typed Arrays in TypeScript

Two equivalent syntaxes (prefer the first):

// ‚úÖ Recommended: type[]
let numbers: number[] = [1, 2, 3, 4, 5];
let names: string[] = ["Ana", "Luis", "Carlos"];
let flags: boolean[] = [true, false, true];

// ‚úÖ Alternative: Array<type> (less common)
let ages: Array<number> = [25, 30, 35];

// ‚ùå TypeScript protects you:
numbers.push("text"); // Error: Argument of type 'string' is not assignable to parameter of type 'number'

3. Type Inference: Let TypeScript Work for You

TypeScript automatically infers types from initial values:

// TypeScript infers number[]
let scores = [10, 20, 30];

// Infers string[]
let cities = ["Madrid", "Barcelona", "Valencia"];

// ‚ö†Ô∏è Be careful with empty arrays:
let items = []; // type: any[] (not safe!)

// ‚úÖ Better: explicitly declare the type
let items: string[] = [];
items.push("item1"); // ‚úÖ Now safe

Rule: If you initialize empty, declare the type. If you initialize with values, let TS infer.

4. Arrays with Multiple Types (Union Types)

Sometimes you need an array to accept multiple types:

// Array that can contain numbers OR strings
let mixed: (number | string)[] = [1, "two", 3, "four"];

// When using them, TypeScript requires type narrowing:
mixed.forEach(item => {
  if (typeof item === "string") {
    console.log(item.toUpperCase()); // ‚úÖ Here item is string
  } else {
    console.log(item.toFixed(2)); // ‚úÖ Here item is number
  }
});

// ‚ö†Ô∏è This doesn't work:
// mixed[0].toUpperCase(); // Error: may be number

5. Tuples: Fixed Position Types

A tuple is an array with a fixed number of elements where each position has a specific type:

// RGB color: [red, green, blue]
let color: [number, number, number] = [255, 128, 0];

// User: [id, username, isActive]
let user: [number, string, boolean] = [1, "ana_dev", true];

// ‚úÖ TS knows the position:
console.log(user[1].toUpperCase()); // "ANA_DEV"

// ‚ùå Errors:
user[0] = "text"; // Error: string is not assignable to number
user = [1, "ana"]; // Error: missing third element

Real use case: React's useState returns a tuple: [value, setter]

6. Arrays of Objects (Most Common in Real Life)

Defining the structure of objects in an array:

// Define the type/interface
type Product = {
  id: number;
  name: string;
  price: number;
  inStock: boolean;
};

// Use it in an array:
let products: Product[] = [
  { id: 1, name: "Laptop", price: 999, inStock: true },
  { id: 2, name: "Mouse", price: 25, inStock: false },
];

// ‚úÖ Autocomplete and type safety:
products.forEach(p => {
  console.log(p.name.toUpperCase());
  console.log(p.price.toFixed(2));
});

// ‚ùå TS catches errors:
products.push({ id: 3, name: "Keyboard" }); // Error: missing 'price' and 'inStock'

7. Array Methods with Automatic Types

TypeScript infers types in callbacks:

let numbers: number[] = [1, 2, 3, 4, 5];

// map: transforms each element
let doubled = numbers.map(n => n * 2); // number[]

// filter: selects elements
let evens = numbers.filter(n => n % 2 === 0); // number[]

// find: finds first matching element
let first = numbers.find(n => n > 3); // number | undefined

// reduce: accumulates a value
let sum = numbers.reduce((acc, n) => acc + n, 0); // number

// ‚úÖ TS knows 'n' is number in all cases, no annotation needed!

Magic: TypeScript automatically infers n is number because numbers is number[].

8. Immutable Arrays: readonly

Prevent modifications to an array:

// Original array (mutable)
let mutableNumbers: number[] = [1, 2, 3];
mutableNumbers.push(4); // ‚úÖ OK

// Immutable array
let immutableNumbers: readonly number[] = [1, 2, 3];
immutableNumbers.push(4); // ‚ùå Error: push doesn't exist on readonly
immutableNumbers[0] = 10; // ‚ùå Error: cannot assign to readonly index

// ‚úÖ But you can read:
console.log(immutableNumbers[0]); // 1
let doubled = immutableNumbers.map(n => n * 2); // ‚úÖ map is allowed (creates new array)

When to use: Function parameters where you shouldn't modify the array, or immutable data in React/Redux.

9. Real Example: Lists in React with TypeScript

How to use typed arrays in a React component:

import { useState } from 'react';

type Todo = {
  id: number;
  text: string;
  completed: boolean;
};

function TodoApp() {
  const [todos, setTodos] = useState<Todo[]>([]);

  const addTodo = (text: string) => {
    const newTodo: Todo = {
      id: Date.now(),
      text,
      completed: false
    };
    setTodos([...todos, newTodo]);
  };

  const toggleTodo = (id: number) => {
    setTodos(todos.map(todo =>
      todo.id === id
        ? { ...todo, completed: !todo.completed }
        : todo
    ));
  };

  return (
    <ul>
      {todos.map(todo => (
        <li key={todo.id} onClick={() => toggleTodo(todo.id)}>
          {todo.text}
        </li>
      ))}
    </ul>
  );
}

Note: useState<Todo[]> tells React that todos is an array of Todo. TypeScript validates everything!

Best Practices Summary

‚úÖ DO

  • ‚Ä¢ Use type[] instead of Array<type>
  • ‚Ä¢ Let TS infer types when initializing with values
  • ‚Ä¢ Declare type for empty arrays: let x: string[] = []
  • ‚Ä¢ Use readonly when you don't want mutations
  • ‚Ä¢ Define type or interface for object arrays

‚ùå DON'T

  • ‚Ä¢ Don't use any[] (loses type safety)
  • ‚Ä¢ Don't mix types without union types
  • ‚Ä¢ Don't forget to narrow types with unions
  • ‚Ä¢ Don't mutate readonly arrays
  • ‚Ä¢ Don't ignore TS errors in .map() / .filter()

Common Errors and Solutions

‚ùå Error: "Type 'string' is not assignable to type 'number'"

let nums: number[] = [1, 2, "3"]; // ‚ùå

Solution: All elements must be numbers:

let nums: number[] = [1, 2, 3]; // ‚úÖ

‚ùå Error: "Property 'toUpperCase' does not exist on type 'number | string'"

let mixed: (number | string)[] = [1, "a"];
mixed[0].toUpperCase(); // ‚ùå

Solution: Use type narrowing:

if (typeof mixed[0] === "string") {
mixed[0].toUpperCase(); // ‚úÖ
}

‚ùå Error: "Cannot assign to readonly array"

let arr: readonly number[] = [1, 2];
arr.push(3); // ‚ùå

Solution: Create a new array:

arr = [...arr, 3]; // ‚úÖ
