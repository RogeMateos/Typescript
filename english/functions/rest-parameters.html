Rest Parameters

Problem

What if a function needs a variable number of arguments?

Solution

Use the ... (rest) operator to capture multiple arguments in an array.

function sum(...numbers: number[]): number {
  return numbers.reduce((a, b) => a + b, 0);
}

sum(1, 2, 3); // 6
sum(5, 10, 15, 20); // 50 (variable number of args)

Important Rules

  • ‚Ä¢Rest is always an array
  • ‚Ä¢Only one rest parameter per function
  • ‚Ä¢Must be the last parameter
  • ‚Ä¢Can receive zero or more arguments

1Ô∏è‚É£ Core Concept: Rest Collects, Spread Expands

The ... operator has two opposite meanings:

  • ‚Ä¢ Rest: Collects multiple values into a single array
  • ‚Ä¢ Spread: Expands a single array/object into multiple values

The difference is context! In parameters = Rest. In calls/literals = Spread.

2Ô∏è‚É£ Rest in Function Parameters

Collect all remaining arguments into an array:

// Collect all numbers into an array
function sum(...numbers: number[]): number {
  return numbers.reduce((a, b) => a + b, 0);
}

sum(1, 2, 3);        // numbers = [1, 2, 3]
sum(10, 20, 30, 40); // numbers = [10, 20, 30, 40]

// Mix regular params with rest
function greet(greeting: string, ...names: string[]): void {
  console.log(greeting + ": " + names.join(", "));
}

greet("Hello", "Alice", "Bob", "Charlie");
// Output: Hello: Alice, Bob, Charlie
‚ö†Ô∏è Rest must be the last parameter:
function bad(age: number, ...names: string[], email: string)
‚ùå Syntax error: rest must be last

3Ô∏è‚É£ Rest ‚â† Optional: Critical Difference

Don't confuse ... (rest) with ? (optional):

// Optional parameter: single value or undefined
function optionalDemo(name?: string): void {
  console.log(name); // string | undefined
}
optionalDemo();        // undefined
optionalDemo("Alice"); // "Alice"
// optionalDemo("Alice", "Bob"); // ERROR!

// Rest parameter: always an array
function restDemo(...names: string[]): void {
  console.log(names); // string[]
}
restDemo();                     // [] empty array
restDemo("Alice");              // ["Alice"]
restDemo("Alice", "Bob", "Charlie"); // ["Alice", "Bob", "Charlie"]
Key difference: Optional gives you `undefined`, Rest gives you an array (possibly empty)

4Ô∏è‚É£ Spread in Function Calls

Expand an array into individual arguments:

function add(a: number, b: number, c: number): number {
  return a + b + c;
}

const numbers = [1, 2, 3];

// Without spread: ERROR (array is single argument)
add(numbers); // ERROR: expects 3 numbers, got 1 array

// With spread: ‚úì Expand array into individual arguments
add(...numbers); // 6 ‚úì

// Real-world example
const arr1 = [1, 2];
const arr2 = [3, 4];
const combined = [...arr1, ...arr2]; // [1, 2, 3, 4]

// Math methods
const numbers = [5, 3, 8, 1];
Math.max(...numbers); // 8
Math.min(...numbers); // 1

5Ô∏è‚É£ Spread in Arrays

Copy and merge arrays without mutation:

// Copy an array (creates new array)
const original = [1, 2, 3];
const copy = [...original]; // [1, 2, 3]

// Merge arrays
const arr1 = [1, 2];
const arr2 = [3, 4];
const merged = [...arr1, ...arr2]; // [1, 2, 3, 4]

// Insert at beginning/middle
const numbers = [2, 3];
const withPrefix = [1, ...numbers, 4]; // [1, 2, 3, 4]

// Clone with modifications
const original = { name: "Alice", age: 30 };
const numbers = [1, 2];
const newNumbers = [...numbers, 3, 4]; // [1, 2, 3, 4]

6Ô∏è‚É£ Spread in Objects (Fundamental in React)

Copy and merge objects without mutation:

// Copy an object (shallow copy)
const user = { name: "Alice", age: 30 };
const copy = { ...user }; // { name: "Alice", age: 30 }

// Merge objects (right overwrites left)
const obj1 = { a: 1, b: 2 };
const obj2 = { b: 3, c: 4 };
const merged = { ...obj1, ...obj2 }; // { a: 1, b: 3, c: 4 }

// Override specific properties
const user = { name: "Alice", age: 30, city: "NYC" };
const updated = { ...user, age: 31 }; // age updated to 31

// Deep merging (only first level)
const original = { user: { name: "Alice", age: 30 } };
const shallow = { ...original }; // Only first level copied
shallow.user.age = 31; // Affects original! (same reference)
‚ö†Ô∏è Note: Spread does shallow copy. Nested objects still reference the original.

7Ô∏è‚É£ Destructuring: Rest vs Spread

Using ... in destructuring (Rest pattern):

// Array destructuring with rest
const [first, second, ...rest] = [1, 2, 3, 4, 5];
// first = 1, second = 2, rest = [3, 4, 5]

// Object destructuring with rest
const { name, age, ...other } = {
  name: "Alice",
  age: 30,
  city: "NYC",
  country: "USA"
};
// name = "Alice", age = 30
// other = { city: "NYC", country: "USA" }

// Skip elements in array
const [, , third, ...rest] = [1, 2, 3, 4, 5];
// third = 3, rest = [4, 5]

8Ô∏è‚É£ React: Flexible Components with Rest + Spread

Pass unknown props to child components:

// Collect all props except 'label'
interface ButtonProps {
  label: string;
  onClick: () => void;
}

function CustomButton({ label, ...rest }: ButtonProps & React.ButtonHTMLAttributes<HTMLButtonElement>) {
  return (
    <button {...rest} className="px-4 py-2 bg-blue-500 text-white rounded">
      {label}
    </button>
  );
}

// Usage: pass any button props
<CustomButton
  label="Click me"
  onClick={() => alert("Clicked")}
  disabled={false}
  type="submit"
/>

The ...rest collects all remaining props, then {...rest} spreads them back to the button element.

9Ô∏è‚É£ React: Immutable State with Spread

Update state without mutations (React best practice):

const [user, setUser] = useState({ name: "Alice", age: 30, city: "NYC" });

// ‚ùå BAD: Direct mutation
user.age = 31; // Doesn't trigger re-render!

// ‚úì GOOD: Create new object with spread
setUser({ ...user, age: 31 });

// Updating nested arrays
const [items, setItems] = useState([
  { id: 1, name: "Item 1", done: false },
  { id: 2, name: "Item 2", done: false }
]);

// Update one item
setItems(items.map(item =>
  item.id === 1 ? { ...item, done: true } : item
));

// Add new item
setItems([...items, { id: 3, name: "Item 3", done: false }]);

// Remove item by id
setItems(items.filter(item => item.id !== 2));

üîü Practical Exercises

Exercise 1: Flexible Sum Function

Create a function that sums numbers passed as arguments:

function sum(...nums: number[]): number {
  return nums.reduce((acc, val) => acc + val, 0);
}

sum(1, 2, 3);              // 6
sum(10, 20, 30, 40, 50);   // 150

Exercise 2: HTML Tag Generator

Create an HTML tag builder using spread and rest:

function createTag(tag: string, content: string, ...attrs: [string, string][]): string {
  const attrStr = attrs.map(([key, val]) => `${key}="${val}"`).join(" ");
  return `<${tag} ${attrStr}>${content}</${tag}>`;
}

createTag("div", "Hello", ["id", "main"], ["class", "container"]);
// <div id="main" class="container">Hello</div>

Exercise 3: Object Merging with Override

Merge multiple objects, with later ones overriding earlier ones:

function merge(...objects: object[]): object {
  return Object.assign({}, ...objects);
  // Or: return objects.reduce((acc, obj) => ({ ...acc, ...obj }), {});
}

const obj1 = { a: 1, b: 2 };
const obj2 = { b: 3, c: 4 };
const obj3 = { c: 5, d: 6 };

merge(obj1, obj2, obj3); // { a: 1, b: 3, c: 5, d: 6 }
