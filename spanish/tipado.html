El Problema del Tipado

Problema

JavaScript es dinámico. Puedes asignar un número a una variable que antes era un texto y el programa no se queja hasta que explota en producción.

Solución

TypeScript añade 'Tipado Estático'. Captura errores mientras escribes, no cuando el usuario navega.

Analogía Visual

"Imagina etiquetar cajas en una mudanza. Sin etiquetas, tienes que abrir todas para saber qué hay (JS). Con etiquetas, sabes exactamente qué esperar sin abrirlas (TS)."

// JavaScript (Peligroso)
function sumar(a, b) {
  return a + b;
}
console.log(sumar('5', 5)); // Resultado: '55'
// TypeScript (Seguro)
function sumar(a: number, b: number): number {
  return a + b;
}
// Error: Argument of type 'string' is not assignable...

Pregunta de Entrevista

Q: ¿Qué es TypeScript y por qué usarlo?

A: Es un superset de JS que añade tipado estático, permitiendo detectar errores en tiempo de compilación y mejorando el autocompletado.
