Null vs Undefined

Problem

JavaScript has two values for "nothing": null and undefined. This causes confusion and errors like "Cannot read property of undefined".

Solution

TypeScript distinguishes between null and undefined, allowing you to control them with strictNullChecks and operators like optional chaining (?.) and nullish coalescing (??).

Visual Analogy

"undefined is a box that was never opened. null is an opened box that is intentionally empty."

// undefined: variable declared but not initialized
let name: string;
console.log(name); // undefined

// null: intentional absence of value
let age: number | null = null;

// With strictNullChecks enabled
let city: string = "Madrid";
// city = null; // ❌ Error: null is not string

// Explicitly allow null
let country: string | null = null; // ✅ OK
country = "Spain"; // ✅ OK
// Optional Chaining (?.)
const user = { name: "Anna", address: undefined };
const street = user.address?.street; // undefined (no error)

// Nullish Coalescing (??)
const port = process.env.PORT ?? 3000;
// Uses 3000 only if PORT is null/undefined (not if it's 0 or "")

// Non-null Assertion (!)
function process(value: string | null) {
  // If you're 100% sure it's not null
  console.log(value!.toUpperCase());
}

// Comparison
console.log(null == undefined);   // true (loose comparison)
console.log(null === undefined);  // false (strict comparison)

Best Practices

  • Enable strictNullChecks in tsconfig.json for better safety
  • Prefer undefined for optional values (TypeScript does this by default)
  • Use null only when you want to indicate intentional absence
  • Use ?. (optional chaining) and ?? (nullish coalescing) for cleaner code
  • Avoid ! (non-null assertion) unless you're completely sure

Interview Prep

Q: What is the difference between null and undefined, and when to use each?

A: undefined means a variable was declared but not initialized (default value). null is an intentional assignment representing "empty". In TypeScript, with strictNullChecks, you must explicitly declare if a type can be null/undefined. Use undefined for optional parameters and null when you want to explicitly indicate "no value".
