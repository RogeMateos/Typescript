Arrays

Problema

En JavaScript, un array puede contener tipos mezclados (números, strings, objetos) causando errores inesperados.

Solución

TypeScript permite definir arrays tipados: todos los elementos deben ser del mismo tipo especificado.

Analogía Visual

"Un estante de libros solo contiene libros, no mezclas con zapatos. Un array tipado solo contiene elementos del tipo declarado."

// Dos formas de declarar arrays tipados
let numeros: number[] = [1, 2, 3, 4, 5];
let nombres: Array<string> = ["Ana", "Juan", "Pedro"];

// TypeScript infiere el tipo
let edades = [25, 30, 35]; // number[]

// numeros.push("texto"); // ❌ Error: string no es number
let frutas: string[] = ["manzana", "pera"];

// Métodos comunes con seguridad de tipos
frutas.push("uva"); // ✅ OK
frutas.map(f => f.toUpperCase()); // ✅ TS sabe que f es string
frutas.filter(f => f.length > 4); // ✅ Autocompletado correcto

// Array de objetos
let usuarios: { nombre: string; edad: number }[] = [
  { nombre: "Ana", edad: 25 }
];

Buenas Prácticas

  • Prefiere number[] sobre Array<number> por legibilidad
  • Define interfaces para arrays de objetos complejos
  • Usa readonly para arrays inmutables: readonly number[]

Pregunta de Entrevista

Q: ¿Cuál es la diferencia entre number[] y Array<number>?

A: No hay diferencia funcional, son sintaxis equivalentes. number[] es más concisa y preferida por la comunidad, mientras que Array<number> es la sintaxis genérica. Ambas definen un array de números.
