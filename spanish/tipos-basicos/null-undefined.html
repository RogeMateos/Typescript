Null vs Undefined

Problema

JavaScript tiene dos valores para "nada": null y undefined. Esto causa confusión y errores como "Cannot read property of undefined".

Solución

TypeScript distingue entre null y undefined, permitiendo controlarlos con strictNullChecks y operadores como optional chaining (?.) y nullish coalescing (??).

Analogía Visual

"undefined es una caja que nunca se abrió. null es una caja abierta que está intencionalmente vacía."

// undefined: variable declarada pero no inicializada
let nombre: string;
console.log(nombre); // undefined

// null: ausencia intencional de valor
let edad: number | null = null;

// Con strictNullChecks activado
let ciudad: string = "Madrid";
// ciudad = null; // ❌ Error: null no es string

// Permitir null explícitamente
let pais: string | null = null; // ✅ OK
pais = "España"; // ✅ OK
// Optional Chaining (?.)
const usuario = { nombre: "Ana", direccion: undefined };
const calle = usuario.direccion?.calle; // undefined (sin error)

// Nullish Coalescing (??)
const puerto = process.env.PORT ?? 3000;
// Usa 3000 solo si PORT es null/undefined (no si es 0 o "")

// Non-null Assertion (!)
function procesar(valor: string | null) {
  // Si estás 100% seguro que no es null
  console.log(valor!.toUpperCase());
}

// Comparación
console.log(null == undefined);   // true (comparación suelta)
console.log(null === undefined);  // false (comparación estricta)

Buenas Prácticas

  • Activa strictNullChecks en tsconfig.json para mayor seguridad
  • Prefiere undefined para valores opcionales (TypeScript lo hace por defecto)
  • Usa null solo cuando quieras indicar ausencia intencional
  • Usa ?. (optional chaining) y ?? (nullish coalescing) para código más limpio
  • Evita ! (non-null assertion) a menos que estés completamente seguro

Pregunta de Entrevista

Q: ¿Cuál es la diferencia entre null y undefined, y cuándo usar cada uno?

A: undefined significa que una variable fue declarada pero no inicializada (valor por defecto). null es una asignación intencional que representa "vacío". En TypeScript, con strictNullChecks, debes declarar explícitamente si un tipo puede ser null/undefined. Usa undefined para parámetros opcionales y null cuando quieras indicar explícitamente "sin valor".
