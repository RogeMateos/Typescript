Never Type: The Impossible

Problem

Some functions never return (throw errors or enter infinite loops). How to represent that something is impossible or will never happen?

Solution

The never type represents values that never occur. It's used for functions that throw errors, infinite loops, or exhaustive validations.

Visual Analogy

"Like a black hole: once you enter, there's no return. Never functions never go back to where they were called."

// Function that throws error (never returns)
function error(message: string): never {
  throw new Error(message);
}

// Infinite loop
function infiniteLoop(): never {
  while (true) {
    console.log("Running...");
  }
}

// TS infers never automatically
function fail() {
  throw new Error("Something went wrong");
  // TS knows this line is unreachable
  console.log("This never executes");
}
// Real use: Exhaustive validation
type Shape = "circle" | "square" | "triangle";

function area(shape: Shape, dimension: number): number {
  switch (shape) {
    case "circle":
      return Math.PI * dimension ** 2;
    case "square":
      return dimension ** 2;
    case "triangle":
      return (dimension ** 2) / 2;
    default:
      // If shape has a new value, TS gives error here
      const _exhaustive: never = shape;
      throw new Error(`Unhandled shape: ${shape}`);
  }
}

// never is the empty union type
type Impossible = string & number; // never

Best Practices

  • Use never in the default of switch for exhaustiveness checking
  • TypeScript infers never automatically in functions that only throw errors
  • never is useful to mark unreachable code or impossible cases
  • Don't confuse never with void: void returns, never doesn't

Interview Prep

Q: When to use never and what is its practical utility?

A: never is used in: 1) Functions that throw errors and never return, 2) Infinite loops, 3) Exhaustive validation in switch statements (if you add a new case to the type and forget to handle it, TS gives an error). It's the "bottom" type that represents impossible values or unreachable code.
