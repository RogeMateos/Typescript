Basic Functions

Problem

Without typing, functions can receive incorrect arguments or return unexpected values, causing silent bugs.

Solution

TypeScript allows typing parameters and return values, ensuring compile-time safety.

Visual Analogy

"A function is like a machine. You need to know what goes in (ingredients) and what comes out (product)."

// Typed function
function greet(name: string): string {
  return `Hello, ${name}!`;
}

const result = greet("John"); // ‚úÖ Correct
greet(123); // ‚ùå Error: expected string

Interview Question

Q: What does a well-typed function look like?

A: Specifying the type of each parameter and the return type.

üìò Tutorial: Function Syntax in TypeScript

1 Basic syntax 

function functionName(parameter: type): returnType {
  // body
  return value;
}

üìå Each parameter must have a type, and the function must indicate what it returns.

2 Return types 

// Returns number
function add(a: number, b: number): number {
  return a + b;
}

// Returns string
function greet(name: string): string {
  return `Hello ${name}`;
}

// Returns boolean
function isAdult(age: number): boolean {
  return age >= 18;
}

3 Arrow functions 

// Arrow function syntax
const add = (a: number, b: number): number => a + b;

// With multiple lines
const multiply = (a: number, b: number): number => {
  const result = a * b;
  return result;
};

Checklist

  • ‚Ä¢Do all parameters have types?
  • ‚Ä¢Is the return type specified?
  • ‚Ä¢Do you always return the correct type?

1 What TypeScript Adds to Functions

TypeScript adds static typing to functions, preventing silent bugs:

  • ‚úÖErrors are caught before running code
  • ‚úÖYour IDE provides better help (autocomplete, refactoring)
  • ‚úÖCode communicates intent better to your team

üëâ Think of types as contracts: "This function promises to receive X and return Y"

2 Ways to Define Functions

Function declaration:

function sum(a: number, b: number): number {
  return a + b;
}

‚úÖ Very clear, ideal for exported utilities

Arrow functions (most used in React):

const subtract = (a: number, b: number): number => {
  return a - b;
};

// Or short version:
const double = (n: number): number => n * 2;

3 Typing Parameters: The Most Important

Without well-typed parameters, the contract is broken from the start.

// ‚úÖ Correct
function greet(name: string): string {
  return `Hello ${name}`;
}

// ‚ùå Incorrect - allows silent errors
function greet(name) {
  return `Hello ${name}`;
}

4 Return Types 

// Basic types
function getHero(): string {
  return "Batman";
}

// void - when you don't care about the return value
const logMessage = (msg: string): void => {
  console.log(msg);
};

// never - when function never terminates normally
const fail = (message: string): never => {
  throw new Error(message);
};

5 Optional, Default, and Rest Parameters 

// Optional parameters
function formatName(name: string, lastName?: string): string {
  return lastName ? `${name} ${lastName}` : name;
}

// Default parameters
function buildUrl(
  path: string,
  base: string = "https://api.myapp.com"
): string {
  return `${base}${path}`;
}

// Rest parameters
function joinWords(...words: string[]): string {
  return words.join(" ");
}

6 Callbacks in React

Typing callbacks prevents errors between components:

type Props = {
  onSave: (value: string) => Promise<void>;
};

function Form({ onSave }: Props) {
  const submit = async () => {
    await onSave("data");
  };

  return <button onClick={submit}>Save</button>;
}

7 Generic Functions (Pro Level) 

function first<T>(arr: T[]): T | undefined {
  return arr[0];
}

// Usage:
first([1, 2, 3]);      // ‚úÖ Infers 'number'
first(["a", "b"]);    // ‚úÖ Infers 'string'

üëâ This is what makes TypeScript truly scalable

Common Pitfalls

Inconsistent returns:

// ‚ùå Returns number | undefined without saying so
function maybeNumber(x: string) {
  if (x === "a") return 1;
}

// ‚úÖ Better
function maybeNumber(x: string): number | undefined {
  if (x === "a") return 1;
  return undefined;
}

Confusing void with undefined:

‚Ä¢ void = "I don't care about the return value"

‚Ä¢ Not the same as returning undefined

Best Practices (Senior Level)

‚úÖ Always Type:

‚ùå Avoid:

  • ‚Ä¢Hidden dependencies
  • ‚Ä¢Abusing inference
