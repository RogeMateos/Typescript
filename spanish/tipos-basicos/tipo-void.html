Tipo Void: Funciones sin Retorno

Problema

Algunas funciones no devuelven un valor útil (como funciones que imprimen en consola o modifican el DOM). ¿Cómo indicar esto en TypeScript?

Solución

El tipo void indica que una función no devuelve ningún valor. Es la forma de TypeScript de decir "esta función hace algo, pero no esperes un resultado."

Analogía Visual

"Como gritar en un valle: haces una acción (gritar) pero no esperas recibir algo tangible de vuelta, solo el eco que no usarás."

// Función con void explícito
function saludar(nombre: string): void {
  console.log(`Hola, ${nombre}`);
  // No hay return (o return sin valor)
}

// Función que modifica el DOM
function actualizarTitulo(titulo: string): void {
  document.title = titulo;
}

// ❌ Error: void no puede asignarse a otro tipo
let resultado: string = saludar("Ana");
// TypeScript infiere void automáticamente
function log(mensaje: string) {
  console.log(mensaje);
  // TS infiere: ': void'
}

// Diferencia con undefined
function retornaUndefined(): undefined {
  return undefined; // Debe retornar explícitamente
}

function retornaVoid(): void {
  return; // Puede no tener return
}

// Callbacks con void
type Callback = (data: string) => void;
const procesar: Callback = (data) => {
  console.log(data);
};

Buenas Prácticas

  • Usa void para funciones que realizan efectos secundarios (logs, DOM, API calls)
  • TypeScript infiere void automáticamente, pero es bueno ser explícito en APIs públicas
  • No confundas void con undefined: void es más flexible en callbacks
  • En callbacks, void permite ignorar el valor de retorno del callback

Pregunta de Entrevista

Q: ¿Cuál es la diferencia entre void y undefined en TypeScript?

A: void significa que el valor de retorno se ignora o no se debe usar (la función puede no tener return). undefined es un tipo y valor específico que debe retornarse explícitamente. En callbacks, void es más flexible: permite que el callback retorne cualquier cosa, pero ese valor será ignorado.
