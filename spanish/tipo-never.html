Tipo Never: Lo Imposible

Problema

Algunas funciones nunca retornan (lanzan errores o entran en bucles infinitos). ¿Cómo representar que algo es imposible o nunca sucederá?

Solución

El tipo never representa valores que nunca ocurren. Se usa para funciones que lanzan errores, bucles infinitos o para validaciones exhaustivas.

Analogía Visual

"Como un agujero negro: una vez que entras, no hay retorno. Las funciones never nunca regresan al punto donde fueron llamadas."

// Función que lanza error (nunca retorna)
function error(mensaje: string): never {
  throw new Error(mensaje);
}

// Bucle infinito
function bucleInfinito(): never {
  while (true) {
    console.log("Ejecutando...");
  }
}

// TS infiere never automáticamente
function fallar() {
  throw new Error("Algo salió mal");
  // TS sabe que esta línea es inalcanzable
  console.log("Esto nunca se ejecuta");
}
// Uso real: Validación exhaustiva
type Forma = "circulo" | "cuadrado" | "triangulo";

function area(forma: Forma, dimension: number): number {
  switch (forma) {
    case "circulo":
      return Math.PI * dimension ** 2;
    case "cuadrado":
      return dimension ** 2;
    case "triangulo":
      return (dimension ** 2) / 2;
    default:
      // Si forma tiene un nuevo valor, TS da error aquí
      const _exhaustivo: never = forma;
      throw new Error(`Forma no manejada: ${forma}`);
  }
}

// never es el tipo de unión vacío
type Imposible = string & number; // never

Buenas Prácticas

  • Usa never en el default de switch para validar exhaustividad
  • TypeScript infiere never automáticamente en funciones que solo lanzan errores
  • never es útil para marcar código inalcanzable o casos imposibles
  • No confundas never con void: void retorna, never no

Pregunta de Entrevista

Q: ¿Cuándo usar never y cuál es su utilidad práctica?

A: never se usa en: 1) Funciones que lanzan errores y nunca retornan, 2) Bucles infinitos, 3) Validación exhaustiva en switch statements (si añades un nuevo caso al tipo y olvidas manejarlo, TS da error). Es el tipo "bottom" que representa valores imposibles o código inalcanzable.
